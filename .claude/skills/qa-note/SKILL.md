你负责把问答内容整理成当前学习系统里的 Obsidian 笔记。

作用范围：
- 只处理当前学习系统内容。
- 如果用户指定了目标文件，优先写入或续写该文件。
- 如果用户没有指定目标文件，优先在当天目录或当前主题目录中查找最近的问答整理笔记；没有则新建。

执行步骤：
1. 先读取 `CLAUDE.md`；如果没有，读取 `CLAUDE.example.md` 和 `AGENTS.md`。
2. 读取 `01-规划规则/笔记与练习题样式约定.md`。
3. 查找最近的问答整理笔记；如果没有样本，使用 `_skeletons/qa-note.md`。
4. 把每个问题整理成一个可折叠 callout，默认格式：

```markdown
> [!note]- 问题标题
> 回答内容
```

5. 如果同一主题下有多个问题，保持同一份笔记里的 callout 风格一致。

约束：
- 不要把问答整理成普通段落堆叠。
- 不要擅自改成新的标题体系。
- 内部链接统一使用 `[[wikilink]]`。
- 如果目标文件已存在，优先续写，不新建重复文件。
- 更新后说明具体更新了哪个文件，以及下一步从哪里继续。

参数：
- `$ARGUMENTS` 可能是主题、目标文件名、目标目录或额外要求。
