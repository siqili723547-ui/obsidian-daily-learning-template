# 每日学习规划要求

每天只回答四件事：

1. 今天继续目标语言的哪个主线主题。
2. 今天最低代码产出是什么。
3. 今天如何判断完成。
4. 明天从哪里继续。

开始每日追踪前，必须先有 [[00-入口/整体学习规划]]。周计划和每日进度只追踪整体规划里的当前语言主线，不临时发明新方向。

整体规划必须适配进度追踪，不能只写“我要学好某某语言”这类宽泛目标。至少要写清：

- 当前阶段的时间范围。
- 当前阶段通过标准。
- 当前阶段最低代码产出。
- 什么算正常进度、轻微落后、明显落后。
- 落后时要补哪些代码产出才能追上。

## 基本约束

- 主线任务控制在 3 个左右。
- 至少留下 1 个可运行或可检查代码产出。
- 至少做 1 次复述、手写、自测或复现。
- 当天没收口，第二天优先收口，不默认开新主题。

## 进入下一主题

满足这些条件再进入下一主题：

- [ ] 有笔记、练习或知识点卡片。
- [ ] 有可运行最小示例或代码练习。
- [ ] 能用自己的话解释核心概念。
- [ ] 复习队列已登记。
