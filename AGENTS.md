# Daily Workspace Rules

这个文件给 Codex / agent 助手读取。

如果你用 Claude，也可以把同样内容放进 `CLAUDE.md`。

## 工作边界

- 只维护当前学习系统目录内的文件。
- 优先续写已有文件，不创建重复副本。
- 不主动删除旧记录。
- 不把未完成主题标记为已掌握。

## 接手顺序

处理学习计划、进度更新、笔记或练习题时，先读：

1. `00-入口/整体学习规划.md`
2. `00-入口/当前学习上下文.md`
3. `00-入口/复习队列.md`
4. 当前周的 `02-周记录/第N周/周计划.md`
5. `01-规划规则/每日学习规划要求.md`
6. `01-规划规则/笔记与练习题样式约定.md`

如果 `整体学习规划.md` 还没填写，先帮助用户补齐整体目标、阶段路线和当前主线，再生成周计划或每日进度。

补整体规划时，必须让规划可追踪，不要只写宽泛愿景。至少补齐：

- 阶段时间范围。
- 阶段通过标准。
- 阶段最低产出。
- 正常、轻微落后、明显落后的判断标准。
- 落后时需要补哪些产出才能追上。

## 启动进度汇报

每次接手或制定当天计划前，先向用户简要汇报：

- 当前整体规划阶段、当前周次和当前主线主题。
- 已完成的关键产出。
- 未完成或卡住的关键产出。
- 复习队列里到期、逾期或缺登记的内容。
- 当前进度判断：正常、轻微落后、明显落后。

如果进度落后，必须明确写出“追上进度还差什么”：

- 还差哪些笔记、练习题、知识点卡片或复习登记。
- 还差哪些可检查产出、自测或复现。
- 今天最小补救动作是什么。
- 哪些任务可以延后，避免为了赶进度同时开多个主题。

## 默认动作

- “制定今天计划”：写入当天 `学习进度.md`。
- “整理笔记”：使用 `_templater/daily-note.md` 的结构。
- “出练习题”：使用 `_templater/daily-exercise.md` 的结构。
- “整理问答”：使用 `_templater/qa-note.md` 的结构，并补复习点。

## 输出要求

每次修改后只说明：

- 更新了哪些文件。
- 下一步从哪里继续。
